# .gitignore Documentation

This document explains what files are ignored by Git and why.

## üîê Environment Variables (CRITICAL)

```gitignore
.env
.env.local
.env.*.local
```

**Why**: Contains sensitive credentials like database passwords, API keys, and service role keys.
**Impact**: NEVER commit these files. They contain secrets that would compromise your security.
**Note**: `.env.example` IS committed as a template (not ignored).

## üìù Logs

```gitignore
logs
*.log
npm-debug.log*
yarn-debug.log*
yarn-error.log*
pnpm-debug.log*
lerna-debug.log*
```

**Why**: Log files are auto-generated and can contain sensitive information or become very large.
**Impact**: No need to version control logs; they're ephemeral.

## üì¶ Dependencies

```gitignore
node_modules
```

**Why**: Node.js dependencies are downloaded via `npm install` based on `package.json`.
**Impact**: Prevents committing 100,000+ dependency files. Required packages are in `package.json`.

## üèóÔ∏è Build Outputs

```gitignore
dist
dist-ssr
.vite
*.local
```

**Why**: Build artifacts are generated from source code during the build process.
**Impact**: Only source code needs versioning; builds are regenerated as needed.

## üß™ Testing

```gitignore
coverage
.nyc_output
*.lcov
```

**Why**: Test coverage reports are auto-generated.
**Impact**: Coverage reports can be regenerated; no need to commit.

## üóÑÔ∏è Database Files

```gitignore
*.sql.backup
*.sql.bak
*.dump
*.sql.gz
*.db
*.sqlite
*.sqlite3
```

**Why**: Local database backups and dumps shouldn't be in version control.
**Impact**: Prevents accidentally committing sensitive data or large database files.
**Note**: `.sql` schema files (like `SUPABASE_SETUP.sql`) ARE committed - these are schemas, not data.

## üìÑ Temporary and Backup Files

```gitignore
*.tmp
*.temp
*~
*.bak
*.swp
*.swo
```

**Why**: Editor and system temporary files.
**Impact**: Keeps repository clean from editor artifacts.

## üíª OS-Specific Files

```gitignore
.DS_Store        # macOS
.DS_Store?       # macOS
._*              # macOS
.Spotlight-V100  # macOS
.Trashes         # macOS
ehthumbs.db      # Windows
Thumbs.db        # Windows
Desktop.ini      # Windows
```

**Why**: Operating system metadata files.
**Impact**: Prevents OS-specific files from cluttering cross-platform projects.

## üñäÔ∏è Editor Directories and Files

```gitignore
.vscode/*                     # VS Code settings
!.vscode/extensions.json     # EXCEPT extensions (committed)
!.vscode/settings.json       # EXCEPT project settings (committed)
!.vscode/tasks.json          # EXCEPT tasks (committed)
!.vscode/launch.json         # EXCEPT launch configs (committed)
.idea                         # JetBrains IDEs
*.suo                         # Visual Studio
*.ntvs*                       # Node.js Tools for VS
*.njsproj                     # Node.js project
*.sln                         # Visual Studio solution
*.sw?                         # Vim swap files
*~                            # Vim/Emacs backup files
```

**Why**: Editor-specific configuration shouldn't force all developers to use the same IDE.
**Impact**: Team members can use their preferred editors.
**Exception**: Shared VS Code settings CAN be committed if needed.

## üìä Script Outputs

```gitignore
script-output.txt
bulk-users-output.log
```

**Why**: Output from custom scripts (like bulk user creation).
**Impact**: Prevents cluttering repository with script execution logs.

## üîí Security Files (CRITICAL)

```gitignore
*.pem              # SSL certificates
*.key              # Private keys
*.cert             # Certificates
credentials.json   # Google/AWS credentials
secrets.json       # Secret configuration
service-account.json  # Service account keys
```

**Why**: Committing these files would expose your infrastructure to attacks.
**Impact**: CRITICAL SECURITY. Never commit these files.
**Note**: If accidentally committed, rotate the keys/certificates immediately.

## ‚úÖ Files That SHOULD Be Committed

These files are NOT ignored and should be committed:

### Configuration
- `.env.example` - Template for environment variables
- `package.json` - Project dependencies
- `package-lock.json` - Locked dependency versions
- `vite.config.js` - Build configuration
- `tailwind.config.js` - Styling configuration
- `eslint.config.js` - Linting rules

### Source Code
- `src/**/*` - All application source code
- `public/**/*` - Static assets

### Database
- `database/*.sql` - SQL schema files (not data dumps)
- `database/*.md` - Documentation
- `database/*.csv` - Test data templates (test_users.csv)

### Scripts
- `scripts/*.js` - Utility scripts

### Documentation
- `README.md` - Project documentation
- `*.md` files - Documentation files

## üö® Security Checklist

Before committing, ensure:

- [ ] No `.env` files in staging area
- [ ] No `node_modules` directory
- [ ] No database dumps (`.sql.backup`, `.dump`)
- [ ] No API keys or credentials
- [ ] No service role keys
- [ ] No private keys or certificates
- [ ] Run `git status` to verify staged files

## üîç Checking What's Ignored

To see what files are being ignored:

```bash
# List all ignored files in repository
git status --ignored

# Check if a specific file is ignored
git check-ignore -v filename.txt

# List all files that would be committed
git ls-files
```

## üõ†Ô∏è If You Accidentally Commit Sensitive Files

If you accidentally commit `.env` or other sensitive files:

1. **Immediately** remove from Git history:
   ```bash
   git rm --cached .env
   git commit -m "Remove sensitive file"
   git push
   ```

2. **Rotate all exposed credentials**:
   - Change database passwords
   - Rotate API keys
   - Generate new service role keys
   - Update `.env` with new credentials

3. **Consider using git-filter-repo** to remove from entire history (advanced)
